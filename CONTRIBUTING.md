# Contributing to OpenWaggle

Thanks for your interest in contributing to OpenWaggle!

## How to Contribute

OpenWaggle is in early development. For now, we accept contributions in the form of **prompt suggestions** — descriptions of issues, features, or improvements that we can implement.

### Suggesting Changes

1. **Open an issue** on [GitHub Issues](https://github.com/OpenWaggle/OpenWaggle/issues) describing the bug, feature, or improvement.
2. Be as specific as possible — include steps to reproduce bugs, expected behavior, or a clear description of the feature you'd like.
3. If relevant, include screenshots or logs.

We review all suggestions and prioritize them for implementation.

### What We're Looking For

- Bug reports with clear reproduction steps
- Feature requests with use cases
- Documentation improvements (typos, unclear instructions, missing content)
- UI/UX feedback

### What We're Not Accepting Yet

- Pull requests with code changes (we'll open this up in the future)
- Large architectural proposals without prior discussion

## Code of Conduct

- Be respectful and constructive in all interactions.
- Focus on the problem, not the person.
- Assume good intent from other contributors.

## Questions?

Open a [GitHub Discussion](https://github.com/OpenWaggle/OpenWaggle/discussions) or reach out via the issue tracker.
