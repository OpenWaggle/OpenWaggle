# Contributing to OpenWaggle

Thanks for your interest in contributing to OpenWaggle! We genuinely appreciate community involvement.

## How to Contribute

OpenWaggle is in active development. We welcome contributions in the form of **issues** — bug reports, feature requests, and improvement ideas are all valuable input that shapes the project.

### Opening Issues

1. **Browse existing issues** on [GitHub Issues](https://github.com/OpenWaggle/OpenWaggle/issues) to see if your idea or bug has already been reported.
2. **Open a new issue** if it hasn't — describe the bug, feature, or improvement clearly.
3. Be as specific as possible — include steps to reproduce bugs, expected behavior, or a clear description of the feature you'd like.
4. If relevant, include screenshots or logs.

We review all issues and track them on our [project roadmap](https://github.com/orgs/OpenWaggle/projects/1).

### What We're Looking For

- Bug reports with clear reproduction steps
- Feature requests with use cases
- Documentation improvements (typos, unclear instructions, missing content)
- UI/UX feedback
- Ideas for new tools, skills, or integrations

### Code Contributions

We're not accepting pull requests with code changes yet — we'll open this up as the project matures. In the meantime, opening a well-described issue is the best way to influence the direction of the project.

Large architectural proposals are welcome as issues for discussion before any implementation.

## Code of Conduct

- Be respectful and constructive in all interactions.
- Focus on the problem, not the person.
- Assume good intent from other contributors.

## Questions?

Open a [GitHub Discussion](https://github.com/OpenWaggle/OpenWaggle/discussions) or reach out via the issue tracker.
