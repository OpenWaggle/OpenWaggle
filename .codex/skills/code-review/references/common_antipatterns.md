# Common Antipatterns

## React and rendering
- Reintroducing manual memoization (`useMemo`, `useCallback`, `React.memo`) without hard evidence in compiler-managed React codebases.
- Running side effects or logging directly inside render paths.
- Deriving query args/keys from `Date.now()` in render, causing data churn and unnecessary refetches.

## Data loading and routing
- Reading request/search params in ad-hoc ways that bypass dependency tracking.
- Leaving dead filter hooks/utilities after route-search state migration.
- Triggering navigation effects while fallback/loading gates should block them.

## Auth and security
- Swallowing auth token fetch failures without structured logging.
- Patching user data from partial callback/webhook payloads without field guards.
- Allowing privileged writes to proceed without role/ban/auth checks.

## Data integrity
- Editing generated artifacts.
- Introducing required schema fields without a migration/cleanup step.
- Reusing unrelated constant groups across domains (blurry invariants, wrong caps).

## Testing and QA
- Using broad `networkidle` waits in E2E tests instead of explicit readiness assertions.
- Adding behavior changes without matching regression tests.
- Relying only on happy-path assertions for async/auth flows.

## Environment and configuration
- Using raw env objects directly instead of validated env accessors.
- Reading `process.env` in client-side runtime paths.
- Committing files generated by tooling when they should remain derived outputs.
